# Análise de Requisitos – Aplicativo Itaú

Este projeto foi desenvolvido como atividade da disciplina **Engenharia de Requisitos**.

O objetivo da atividade foi analisar um sistema digital real sob a perspectiva da **Engenharia de Requisitos**, identificando funcionalidades, características de qualidade e possíveis falhas na definição ou implementação de requisitos.

## 📱 Sistema analisado

Aplicativo do **Banco Itaú**

O sistema foi escolhido por ser uma plataforma amplamente utilizada no Brasil para serviços bancários digitais, permitindo analisar funcionalidades reais e problemas observados no uso cotidiano.

## 🎯 Objetivo da Atividade

Desenvolver a capacidade de:

* Identificar **requisitos funcionais**
* Identificar **requisitos não funcionais**
* Analisar possíveis **falhas de requisitos**
* Propor **melhorias para o sistema**
* Refletir sobre problemas relacionados à **definição ou implementação de requisitos**

## 📄 Estrutura da Análise

A análise realizada no trabalho contempla:

1. **Requisitos Funcionais**
   Identificação de funcionalidades principais presentes no sistema.

2. **Requisitos Não Funcionais**
   Características de qualidade esperadas para o funcionamento adequado do aplicativo.

3. **Possíveis Falhas de Requisitos**
   Problemas observados ou potenciais falhas relacionadas à usabilidade, desempenho ou clareza das funcionalidades.

4. **Sugestões de Melhoria**
   Propostas de melhorias baseadas em conceitos da Engenharia de Requisitos.

5. **Reflexão Crítica**
   Análise sobre se os problemas identificados estão relacionados a falhas de implementação técnica ou falhas na definição dos requisitos.
